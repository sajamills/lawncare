## Codebase Patterns
- Next.js 15 project using App Router with `src/` directory layout (`src/app/`, `src/lib/`, `src/data/`, `src/actions/`)
- Tailwind CSS v4 uses `@import "tailwindcss"` in globals.css (not `@tailwind` directives)
- Tailwind CSS v4 custom colors: define via `@theme { --color-<name>: <value>; }` in globals.css; no tailwind.config.ts needed
- Use `var(--color-<name>)` in inline styles or CSS for semantic color tokens defined in `@theme`
- TypeScript strict mode enabled via tsconfig.json
- .gitignore blocks `.env*` — commit example env files as `env.local.example` (no leading dot)
- Clerk auth: clerkMiddleware + createRouteMatcher in src/middleware.ts; ClerkProvider wraps layout; server components use `auth()` from @clerk/nextjs/server
- Onboarding state persisted in sessionStorage under key `onboarding_state`; session UUID in localStorage under `lawn_session_id`
- All onboarding step components take `{ onNext: () => void }` prop; routing handled by `src/app/onboarding/[step]/page.tsx`
- Extension PDF sources in `src/data/extension-sources.ts`: `extensionSources[STATE]` gives university info; `knownPdfUrls[STATE_grassType]` gives direct PDF URL
- DB uses `@vercel/postgres` (createPool); works with any `POSTGRES_URL` env var including Supabase connection strings
- Warning colors: background #3d3000, text #eab308 (used for pet safety notes — reuse for pre-emergent banners)
- grassTypes data lives in `src/data/grass-types.ts`; each entry has id, name, description, traits[], photoUrl, photoCredit
- `/api/parse-pdf` route: accepts { state, grassType, pdfUrl? }; checks DB cache first; calls Claude claude-sonnet-4-6 to generate plan; caches result in cached_plans table
- ISO week utility: getISOWeek(date) using UTC math — always use for week-of-year calculations to avoid timezone bugs
