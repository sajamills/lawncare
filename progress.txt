## Codebase Patterns
- Next.js 15 project using App Router with `src/` directory layout (`src/app/`, `src/lib/`, `src/data/`, `src/actions/`)
- Tailwind CSS v4 uses `@import "tailwindcss"` in globals.css (not `@tailwind` directives)
- Tailwind CSS v4 custom colors: define via `@theme { --color-<name>: <value>; }` in globals.css; no tailwind.config.ts needed
- Use `var(--color-<name>)` in inline styles or CSS for semantic color tokens defined in `@theme`
- TypeScript strict mode enabled via tsconfig.json
- .gitignore blocks `.env*` — commit example env files as `env.local.example` (no leading dot)
- Clerk auth: clerkMiddleware + createRouteMatcher in src/middleware.ts; ClerkProvider wraps layout; server components use `auth()` from @clerk/nextjs/server (currently in demo/bypass mode)
- Onboarding state persisted in sessionStorage under key `onboarding_state`; session UUID in localStorage under `lawn_session_id`
- All onboarding step components take `{ onNext: () => void }` prop; routing handled by `src/app/onboarding/[step]/page.tsx`
- Extension PDF sources in `src/data/extension-sources.ts`: `extensionSources[STATE]` gives university info; `knownPdfUrls[STATE_grassType]` gives direct PDF URL
- DB uses `@vercel/postgres` (createPool); works with any `POSTGRES_URL` env var including Supabase connection strings
- Warning/alert colors: background #3d3000, text #eab308 (used for pet safety notes and pre-emergent banners)
- grassTypes data in `src/data/grass-types.ts`; each entry has id, name, description, traits[], photoUrl, photoCredit
- Pre-emergent windows in `src/data/preemergent-windows.ts`; getPreemergentWindow(state) returns spring/fall date ranges
- `/api/parse-pdf` route: accepts { state, grassType, pdfUrl? }; checks DB cache first; calls Claude claude-sonnet-4-6; returns WeeklyPlan[] (52 entries); WeeklyPlan and WeeklyTask types are exported from that route file
- ISO week utility getISOWeek(date): uses UTC math; week 1 = week containing Jan 4
- Pexels CDN URL format: https://images.pexels.com/photos/[ID]/pexels-photo-[ID].jpeg?auto=compress&cs=tinysrgb&w=640
- next.config.ts has PWA wrapper in prod — add config changes to base nextConfig object before the PWA wrap
- Bracket paths require quoting in git add: git add "src/app/onboarding/[step]/..."
- Profile page at src/app/profile/page.tsx: full edit form (ZIP, grass type, sq ft, pets, sun); uses saveProfile() and getProfile() from @/actions/profile
- Calendar page at src/app/dashboard/calendar/page.tsx: CURRENTLY BROKEN — still uses old MonthlyPlan (12-month) interface; needs rewrite for WeeklyPlan (52-week)
- Dashboard at src/app/dashboard/page.tsx: reads onboarding_state from sessionStorage; calls /api/parse-pdf; uses getISOWeek to find current week tasks
- Layout at src/app/layout.tsx: header has only LawnGuide wordmark; NO nav links exist yet
