## Codebase Patterns
- Next.js 15 project using App Router with `src/` directory layout (`src/app/`, `src/lib/`, `src/data/`, `src/actions/`)
- Tailwind CSS v4 uses `@import "tailwindcss"` in globals.css (not `@tailwind` directives)
- Tailwind CSS v4 custom colors: define via `@theme { --color-<name>: <value>; }` in globals.css; no tailwind.config.ts needed
- Use `var(--color-<name>)` in inline styles for semantic color tokens defined in `@theme`
- TypeScript strict mode enabled via tsconfig.json
- .gitignore blocks `.env*` — commit example env files as `env.local.example` (no leading dot)
- Clerk auth: @clerk/nextjs package; server actions use auth() from @clerk/nextjs/server; client components use useUser()/useClerk() hooks; ClerkProvider wraps layout
- Onboarding state persisted in sessionStorage under key `onboarding_state`; session UUID in localStorage under `lawn_session_id`
- All onboarding step components take `{ onNext: () => void }` prop; routing handled by `src/app/onboarding/[step]/page.tsx`
- DB uses `@vercel/postgres` (createPool); works with any `POSTGRES_URL` env var; createPool() is LAZY (only called on first query) to avoid module-load crash in dev without POSTGRES_URL
- All DB migrations use IF NOT EXISTS — safe to re-run
- grassTypes data in `src/data/grass-types.ts`; each entry has id, name, description, traits[], photoUrl, photoCredit, bladeWidth, texture, activeSeason
- Pre-emergent windows in `src/data/preemergent-windows.ts`; getPreemergentWindow(state) returns spring/fall date ranges
- `/api/parse-pdf` route: accepts { state, grassType, pdfUrl? }; checks DB cache first; calls Claude claude-sonnet-4-6; returns WeeklyPlan[] (52 entries); WeeklyPlan and WeeklyTask types exported from that route file
- ISO week utility getISOWeek(date): uses UTC math; week 1 = week containing Jan 4
- Pexels CDN URL format: https://images.pexels.com/photos/[ID]/pexels-photo-[ID].jpeg?auto=compress&cs=tinysrgb&w=640 — ALWAYS verify photo ID returns 200 before using
- next.config.ts has PWA wrapper in prod — add config changes to base nextConfig object before the PWA wrap
- Bracket paths require quoting in git add: git add "src/app/onboarding/[step]/..."
- Layout: src/app/layout.tsx is a SERVER component (exports metadata); client nav is in src/app/ClientNav.tsx ("use client", usePathname, nav rendering)
- Design tokens in globals.css @theme: --color-background, --color-surface, --color-primary, --color-secondary, --color-text-primary, --color-text-muted, --color-surface-alt, --color-border, --color-warning-*, --color-urgent, --color-routine, --color-optional
- Hardcoded hex colors to AVOID (use CSS vars instead): #2d4a2d → var(--color-border), #3d3000 → var(--color-warning-bg), #eab308 → var(--color-warning-text), #1a3a1a → var(--color-surface-alt)
- SVG icon components: src/components/icons.tsx exports HomeIcon, CalendarIcon, ProfileIcon
- Always use borderWidth + borderStyle as separate inline style props when you need consistent border width regardless of selection state (avoids layout shift)
- Stripe webhooks: MUST use request.text() for raw body (NOT request.json()) before calling stripe.webhooks.constructEvent()
- Vercel Blob: requires BLOB_READ_WRITE_TOKEN; use put(pathname, body, { access: 'public' }) from @vercel/blob
- Claude Vision: messages array includes { type: 'image', source: { type: 'url', url: imageUrl } } image block; model claude-sonnet-4-6
- Google Maps: use @googlemaps/js-api-loader Loader singleton; libraries: ['places', 'drawing', 'geometry']; mapTypeId: 'satellite' with tilt: 0 for flat satellite view
- Deploy after EVERY story: run `vercel --yes` and verify on production URL before marking passes: true
